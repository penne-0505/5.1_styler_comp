# 使用ガイド (Usage Guide)

このプロジェクトは、LLMの異なるスタイラー（ChatGPT 5.1 thinking styleなど）による出力の差異を検証・比較するためのものです。

## プロジェクト構成

```
5.1_styler_comp/
├── inputs/              # テスト用の入力ファイル（設問）
│   ├── input_001.txt
│   ├── input_002.txt
│   └── ...
├── outputs/             # 各スタイラーの出力結果
│   ├── styler_a/
│   │   ├── output_001.txt
│   │   └── ...
│   ├── styler_b/
│   │   └── ...
│   └── styler_c/
│       └── ...
├── comparisons/         # スタイラー間の比較結果
│   ├── comparison_001.md
│   └── ...
├── docs/                # プロジェクトドキュメント
│   ├── usage_guide.md
│   └── styler_specifications.md
└── README.md
```

## ワークフロー

### 1. 入力ファイルの準備

`inputs/` ディレクトリに新しいテストケースを追加します。

```bash
# 例: 新しい入力ファイルを作成
echo "あなたの質問内容" > inputs/input_004.txt
```

### 2. 各スタイラーで実行

各スタイラーを使用してLLMを実行し、結果を対応するディレクトリに保存します。

```bash
# 例: Styler Aでの出力を保存
# [LLMツールを使用して出力を生成]
# 出力を outputs/styler_a/output_004.txt に保存
```

### 3. 出力の保存

生成された出力を適切なディレクトリに保存します：

- Styler A → `outputs/styler_a/output_XXX.txt`
- Styler B → `outputs/styler_b/output_XXX.txt`
- Styler C → `outputs/styler_c/output_XXX.txt`

**重要**: 出力ファイル名は対応する入力ファイルの番号と一致させてください。

### 4. 比較と分析

異なるスタイラーの出力を比較し、`comparisons/` ディレクトリに結果を記録します。

```markdown
# comparisons/comparison_004.md の例

## 入力: input_004
[質問内容]

## Styler Aの出力
[要約と評価]

## Styler Bの出力
[要約と評価]

## Styler Cの出力
[要約と評価]

## 総合評価
[比較結果と推奨事項]
```

### 5. サマリーの更新

定期的に `comparisons/summary.md` を更新し、全体的な傾向や発見事項を記録します。

## ベストプラクティス

1. **一貫性のある命名**: 入力と出力のファイル番号を一致させる
2. **詳細な記録**: 各比較で具体的な観察結果を記録する
3. **定期的な評価**: 複数のテストケースを実行した後、パターンを分析する
4. **ドキュメント化**: 発見した重要な差異や特徴を文書化する

## 評価の観点

各スタイラーの出力を評価する際は、以下の観点を考慮してください：

- **正確性**: 情報の正確さと妥当性
- **完全性**: 質問への網羅的な回答
- **明確性**: 理解しやすい説明
- **構造**: 論理的な構成
- **スタイル**: 文体とトーン
- **実用性**: 実際の使用における有用性

## トラブルシューティング

### 出力ファイルが見つからない
- ファイル名が正しいか確認してください（`output_XXX.txt`）
- 対応するスタイラーディレクトリに配置されているか確認してください

### 比較が困難
- 同じ入力に対する複数のスタイラーの出力が揃っているか確認してください
- ファイル番号が一致しているか確認してください

## 追加のスタイラーの追加

新しいスタイラーを追加する場合：

```bash
# 1. 新しいスタイラーディレクトリを作成
mkdir outputs/styler_d

# 2. READMEファイルを作成してスタイラーの特徴を記述
cat > outputs/styler_d/README.md << EOF
# Styler D
[スタイラーの説明]
EOF

# 3. 既存の入力に対する出力を生成して保存
```
