## 全体のイメージ（確実）

まずざっくり言うと、HTTP と HTTPS は「インターネット上で、パソコンやスマホがサーバーと会話するときのルール（約束事）」です。
どちらも「Web ページをください」「このボタンが押されました」などの情報をやり取りするしくみですが、HTTPS はその会話に「鍵つきの安全対策」がついたバージョンだと考えてください。

よくある説明として、HTTP は「文字が丸見えのハガキ」、HTTPS は「中身が見えない鍵つきの封筒」のようなものだと言われます。このイメージを頭に置いたまま読み進めると理解しやすくなります。

---

## HTTP のしくみ（ざっくり）（確実）

HTTP は「HyperText Transfer Protocol（ハイパーテキスト転送プロトコル）」という名前で、ブラウザ（Chrome や Edge など）と Web サーバーがやり取りするための基本ルールです。

ブラウザが「このページを見せて」とサーバーにお願いし、サーバーが「このページだよ」と HTML や画像などを返します。この会話の中身は、そのままのテキストに近い形でネットを流れます。

つまり HTTP では、

* 何のページを開こうとしているか
* 入力フォームに何を書いたか（ID やパスワード、住所など）

といった情報が、そのままネットの途中を通り抜けていくイメージになります。途中を通る人（Wi-Fi の持ち主や悪意のある人）が本気でのぞき見しようとすると、内容が読めてしまう可能性があります。

---

## HTTPS のしくみ（ざっくり）（確実）

HTTPS は「HTTP に S（Secure＝安全）」がついたもので、正式には「HTTP over TLS/SSL」と呼ばれます。ここでいちばん大事なのは「暗号化」と「相手確認」の二つです。

まず「暗号化」です。
ブラウザとサーバーの間で、最初に「この鍵を使ってお互い暗号をかけようね」という話し合いをします。ここで使う鍵はコンピュータが自動で作る数字の組み合わせです。この鍵を使って、送るデータをぐちゃぐちゃに変えてからネットに流します。途中でデータをのぞき見されても、「意味の分からない記号の列」にしか見えません。

次に「相手確認（認証）」です。
本物の Web サイトであることを証明するために、「証明書」と呼ばれるものを使います。証明書は「このサイトは本当にこの会社が運営していますよ」と第三者の機関が確認してくれる仕組みです。ブラウザは証明書をチェックして、「怪しくないサイトか」を確かめます。アドレスバーの左に鍵マークが出るのは、この確認がうまくいったときです。

このように HTTPS では、

* 通信内容が暗号化される
* 相手のサイトが本物かどうか確認される

という二つの安全対策が入っています。

## なぜ HTTPS が使われるのか（確実〜高確度）

HTTPS が使われる主な理由は、次のような危険を減らすためです。ここで挙げる理由は、インターネットの安全設計で一般的に認められている考え方なので、確実〜高確度の説明になります。

1つ目は「のぞき見を防ぐため」です。
HTTP だと、フリーワイファイなどを通したときに、同じネットワークにいる人が特別なソフトを使うと、あなたがどんな情報を送ったかを読み取れてしまう可能性があります。HTTPS だと、その情報が暗号化されているため、読み取れても内容が分かりません。

2つ目は「なりすましを防ぐため」です。
見た目がそっくりなニセサイトを作って、そこに ID やパスワードを入力させようとする攻撃があります。HTTPS の証明書を正しくチェックすることで、「本当にその会社のサーバーなのか」を確かめることができ、なりすましの一部を防ぐことができます。

3つ目は「改ざんを防ぐため」です。
途中で誰かがデータを書き換えて、勝手に別の広告を差し込んだり、ウイルスを仕込んだりする攻撃もあります。HTTPS では「データが途中で書き換えられていないか」をチェックする仕組みも含まれているため、改ざんに気づくことができます。

このような理由から、今は多くの Web サイトが HTTPS に対応しており、ブラウザ側も「HTTP だけのサイト」には警告を出すことが増えています。

## 実生活の例で考えてみる（高確度）

ここではイメージしやすいように、郵便のやり取りにたとえて考えてみます。たとえなので、細かい部分は現実と少し違いますが、全体のイメージとしては高確度で有効な説明です。

HTTP は、さきほど触れた通り「ハガキ」のようなものです。
ハガキには、そのまま文字が書いてあります。あなたが誰かに住所やメッセージを書いて送るとき、郵便局の人や、途中でハガキを見た人には、書いてある内容が丸見えです。普通のあいさつ程度なら問題ないかもしれませんが、お金やパスワードなど大事なことを書くにはあまり向きません。

一方、HTTPS は「鍵付きの箱や、しっかり封をした封筒」のようなものです。
手紙は封筒の中に入っていて、外からは中身が見えません。さらに鍵付きの箱だと、箱を開けられるのは鍵を持っている人だけです。途中で誰かが箱を開けて中身を見ようとしても、鍵がないので簡単には開けられません。箱が壊されていたらすぐ分かるので、「途中で何かあったな」と疑うこともできます。

インターネットの世界では、あなたのスマホやパソコンが「手紙を出す人」、Web サイトのサーバーが「手紙を受け取る人」にあたります。HTTP はハガキで会話している状態、HTTPS は鍵付きの箱で会話している状態だと考えると、どちらが安全そうかはイメージしやすいはずです。

---

## まとめ（確実）

HTTP と HTTPS はどちらも「Web ページを見るための会話のルール」ですが、HTTP ではやり取りの中身がそのままネットを流れ、HTTPS では暗号化と相手確認によって安全性が高くなります。この違いによって、のぞき見・なりすまし・改ざんといった問題を減らせるため、今では多くのサイトが HTTPS を使うようになっています。
